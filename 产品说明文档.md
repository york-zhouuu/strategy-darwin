# strategy-darwin · 策略的进化机器

> **AI 时代,人人都能让 AI 写出一个"年化 50%"的策略。**
> **没有人告诉你:那个数字,大概率是 AI 编的。**
> 我们造了一台不会骗你的机器。

---

## 1. 项目名称

**strategy-darwin(策略的进化机器)**
像达尔文的自然选择一样运转:AI 负责大量产生策略想法(**变异**),一台铁面无私的审判机器负责淘汰(**选择**),每个死掉的策略都留下死因、喂养下一代(**遗传**)。

## 2. 项目简介

这是一台**自己会转的策略研究机器**:你丢进去一个交易想法——一句话、一篇论文、甚至什么都不丢让它自己挖——它会自动完成实现、检验、判决,然后把"为什么死"变成下一个更好的想法。

它和市面上所有"AI 炒股"产品有一个根本区别:

**别人向你展示赚钱的回测;我们向你展示 95% 的策略是怎么死的。**

因为前者几乎总是幻觉,而后者才是真的。这台机器存在的意义只有一句话:**当它最终对某个策略说"通过",那句话是可以信的。**

## 3. 它解决什么问题

量化研究最大的敌人,从来不是"找不到策略",而是**自己骗自己**。AI 让这件事严重了十倍,因为出现了三种新型骗局:

**骗局一:AI 在背答案。**
大语言模型读过整个互联网——包括"历史后来发生了什么"。让 AI 回测历史策略,它不是在预测,是在**默写**。回测 100 分,实盘裸奔。

**骗局二:考完试再改及格线。**
人和 AI 都一样:看到结果不理想,就忍不住换个参数、换个时间段、换个标准,直到"显著"为止。试的次数够多,垃圾也会闪光。

**骗局三:纸面利润,永远买不到。**
回测里的价格,等你真去下单时早就不在了;点差、手续费、流动性,能把"每年 30%"吃成负数。无数策略死在"看得见、吃不着"。

普通人识别不了这三种骗局,大部分专业工具也不管——它们的 KPI 是"帮你找到好看的回测"。**strategy-darwin 的 KPI 相反:确保好看的回测骗不了你。**

## 4. 核心功能(用户视角)

**你只需要做一件事:把想法丢进来。** 在 Claude Code 里,四个命令就是整个产品:

| 你说 | 机器做 |
|---|---|
| `/strategy 财报后股价会漂移,买它` | 把这句话变成可检验的实验,全流程审判,给你判决书 |
| `/paper-to-factor <一篇论文>` | 把论文里所有策略挖出来,逐个送审 |
| `/strategy-loop` | 它自己想、自己测、自己迭代,机会来了叫你 |
| `/strategy-status` | 仪表盘:测过什么、死于什么、谁还活着 |

每个想法进来后,都要过一条**不可跳关的审判流水线**:

1. **先立字据**——检验开始前,通过标准就被"公证"锁死。事后想改标准?机器直接拒绝执行。
2. **AI 测谎**——把资料全部抽走只留股票代码,AI 还能"预测"对?那它是在背答案,判决作废。这是我们独创的、针对"骗局一"的测谎仪。
3. **时间体检**——很多策略只是"前几年赚过";机器强制检查**最近一段还灵不灵**,专治"吃老本的幽灵策略"。
4. **落地体检**——模拟你真实下单:晚一天买还赚吗?算上全部成本还赚吗?你的资金量塞得进去吗?"看得见吃不着"的,一律拦下。
5. **判决与遗书**——绝大多数策略在这里死掉,但每一个都留下**死因报告**:是背答案?吃老本?还是买不着?这份"遗书"自动成为下一代策略的进化方向。

连机器自己也被监督:所有被判死刑的策略会**继续在纸面上模拟运行**,定期回头检查"有没有错杀好人"——审判者也要接受审判。

## 5. AI 用在哪(方式与方案)

一句话:**AI 是这台机器里最能干的员工,但永远不是法官。**

- **AI 当研究员**:阅读论文、结合历史失败教训,提出下一个策略假设(用了 Claude 智能体工作流 + 提示词工程 + 跨会话记忆);
- **AI 当工程师**:把自然语言想法在几分钟内变成可运行的实验代码(Claude Code 智能体 + 工具调用 + 自动化工作流);
- **AI 当红队**:独立的 AI 分身专门负责攻击"通过"的判决,找茬(多智能体对抗);
- **但 AI 永远不当法官**:所有判决由确定性的程序做出——统计检验、冻结的标准、测谎探针。AI 说不动它,也绕不过它。

**本作品最独特的 AI 观点:AI 既是生产力,也是嫌疑人。** 我们可能是极少数给 AI 配了测谎仪的量化项目——因为在金融历史数据上,AI"过目不忘"恰恰是它最危险的地方。

(工程上:Claude Code 承载全部智能体流程;实验中另用 DeepSeek/豆包 API 做逐事件判断;判决引擎为自研纯 Python,零第三方依赖。)

## 6. 数据来源与处理

全部使用**公开数据**,已随仓库附带(~36MB),不含任何个人信息、内幕信息或受限数据:

- 预测市场 **7,718 个已结算市场**的价格历史(Polymarket 公开 API,2022–2026)
- 美股 **235 只股票**日线(Stooq/Nasdaq 公开接口,2020–2026)
- 加密货币 **145 个币种**行情(Binance 公开 API,2023–2026)

处理方式为标准的事件研究(时点对齐、除权调整、异常值处理),文档中如实标注了已知局限(如美股样本的幸存者偏差——它只会让我们的审判更严格,不会更宽松)。

## 7. 怎么体验(评委 60 秒路径)

**零部署、零账号、零 API key、零第三方依赖**——装了 Python 的电脑就能跑:

```bash
git clone https://github.com/york-zhouuu/strategy-darwin && cd strategy-darwin

python3 src/valkit/lane.py       # 看点①:想跳过公证、想事后改标准 → 机器当场拒绝
python3 src/valkit/leakage.py    # 看点②:一个回测满分的"天才策略"被测谎仪当场揭穿:它在背答案
python3 scripts/run_valkit_pm.py # 看点③:7718个真实市场,完整审判一个真策略 → 判死 + 两条死因
```

配套录屏 Demo 演示同一路径 + Claude Code 驾驶舱实操。

## 8. 创新点

1. **给 AI 配测谎仪**:业内都在担心"大模型回测作弊",我们把这个担心做成了可运行的检测——抽走资料还答对 = 在背答案,当场揭穿。
2. **法庭不许翻案**:通过标准在看到结果之前就被程序锁死,事后修改会被**代码层面拒绝**——把科研界的"预注册"纪律,第一次做成了量化工具里不可绕过的机关。
3. **死亡即养分**:每个被杀死的策略自动留下死因,直接变成下一代策略的进化方向——验证器同时是创意引擎,这台机器越用越聪明。
4. **审判者也受审**:被判死刑的策略继续纸面陪跑,定期审计"有没有错杀"——连机器自己的偏见都被量化和纠正。开发期间,它已经两次抓出**人工分析漏掉的策略死因**、两次发现并修复了**自己审判标准的漏洞**,全程有账本记录。

## 10. 风险提示与边界

我们想说得比任何人都直白:

- **这台机器不承诺帮你赚钱;它承诺不骗你。** 演示中所有策略都被判死——这不是产品失败,这恰恰是它在正确工作:市场上绝大多数"策略"本来就该死。
- 它把"哪天真的出现一个能通过的策略"这件事,变得**可以被相信**;
- 已知局限:演示数据存在幸存者偏差(使审判偏严格)、真正的最终确认需要时间(快不了,这是特性不是缺陷);
- 本作品为研究工具,**不构成任何投资建议**。
