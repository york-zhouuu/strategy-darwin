# 产品说明文档

## 1. 项目名称

**strategy-darwin** —— 策略的进化机器(变异×选择×遗传)
核心裁判引擎名 **valkit**;口号:回测只是上界,当机器说 GO,那句话才是真的。

## 2. 项目简介

一台把**自然选择**搬进量化研究的自迭代策略机器:LLM 生成策略假设(**变异**),工具强制的证伪流水线杀掉 ~95% 经不起检验的、评级层再拦住"真但吃不到"的(**选择**),机制库把每次死亡的死因与洞察传给下一代假设、教训还会升级为新闸门——选择压自身也在进化(**遗传**)。全程 append-only 账本可审计。入口是 Claude Code 里的四个斜杠命令(驾驶舱),LLM 兼任研究员与工程师,但**判决永远由确定性工具做出,LLM 绕不过**。

## 3. 解决的问题 / 应用场景

个人量化研究最大的敌人不是找不到信号,而是**系统性自欺**,LLM 时代更甚:

| 自欺方式 | 后果 |
|---|---|
| p-hacking(看完结果改门槛/换参数直到"显著") | 假信号上实盘,真金白银亏损 |
| **LLM 泄漏**(模型权重记得历史结局,回测"预测"实为背题) | AI 量化回测普遍虚高,不可信 |
| 幸存者偏差/买卖价差弹跳/成本幻觉 | "对但吃不到"的策略被误判为可交易 |
| 老年份撑起混池均值(edge 早被套利磨平仍显著) | 追一个已死的信号 |

场景:个人/小团队($10k–100k 量级)的策略研究与验证;论文/研报因子的批量复现审查;任何"AI 生成策略"流程的可信性把关。

## 4. 核心功能

**① 杀戮道(是不是真的)** —— 强制流水线 `KillLane`,五道不可跳的闸:
- **冻结预注册**:看任何结果之前钉死判定门槛并哈希入账本;之后 register 会被**拒绝执行**(防 retrofit);
- **仪器正控**:先向合成数据注入已知信号确认引擎测得回、喂纯噪声确认不误报,正控不过拒绝出判决;
- **强制检验组**:全样本/分桶/**时间衰减切片**(抓"老年份撑混池")/样本内外切分,一个不能少;
- **agent 三闸**(策略含 LLM 判断时):泄漏探针(无输入/反事实占位,量化"背题"底噪)、模型 cutoff 窗、PIT 时间戳校验——任一污染够不到 GO;
- **可交易性四检**:尾部驱动/做空依赖/**入场时点弹跳**/成本吞噬——"判断准但吃不到"是最常见死法。

**② 确认道(能不能真做 + 引擎有没有漏)**:
- **致命闸评级** `grade`:全口径成本(含资金费/滑点)/容量/可做空性/爆仓风险/regime 一致性/运营,分级 kill / 长期观察 / 重点观察;
- **影子登记+召回审计** `paper`:GO 和 KILL **都**前向登记(写入时对结局全盲,结算物理隔离),定期按死因聚类审计"被误杀的策略"反哺引擎。

**③ 驾驶舱(Claude Code 四个斜杠命令)**:`/strategy`(自然语言想法→全流水线)、`/strategy-loop`(自挖掘自迭代,带 shot 预算与停机条件)、`/paper-to-factor`(论文→可测因子 spec)、`/strategy-status`(仪表盘)。

## 5. AI 使用方式与技术方案

AI 在本作品中**既是核心生产力,又是被审计对象**——这是与常见"AI 炒股"项目的根本区别:

- **AI 作为研究员+工程师(核心功能)**:Claude Code agentic 工作流承担 R(从机制库/失败诊断生成下一个假设)与 D(自然语言/论文 → 适配器代码);通过 **skills(提示词工程)** 把纪律工作流固化为可复用命令;**subagent** 用作独立红队对 GO 判决做对抗审查;跨 session **memory** 承载机制库(确认/证伪/洞察的累积)。
- **AI 作为被审计对象(创新核心)**:凡决策链含 LLM 定性判断的策略,回测结论一律只当**性能上界**;用**对抗性泄漏探针**(抽掉输入还准=在背题;喂假数据还准=在漏)量化 LLM 的"权重前视"底噪并从表观 edge 中扣除。
- **工具调用/自动化工作流**:判决、统计(符号检验/Bonferroni/时间切片)、冻结哈希、账本全部由确定性 Python 工具执行,**LLM 无权改判**。
- 实验中还用到 DeepSeek / Doubao API 做逐事件定性判断与假设生成(完整工作区,演示仓库不需要 API key)。
- 设计灵感部分来自 Microsoft RD-Agent(MIT),取其"R+D 分工/因子抽取链",**弃其"回测数字驱动迭代"的反馈环**(该反馈环本质是 p-hacking 自动机;拆解记录在 `docs/RD-Agent拆解-可搬资产.md`)。

## 6. 数据来源与处理方式

全部为**公开数据**,仅用于研究/演示,已随仓库附带(共 ~36MB):

| 数据 | 来源 | 范围 | 用途 |
|---|---|---|---|
| 预测市场 7718 个已结算市场日级价格 | Polymarket Gamma/CLOB 公开 API | 2022–2026 | fade 策略端到端验证 |
| 美股日线 235 票(仅收盘价) | Stooq/Nasdaq 公开接口 | 2020–2026 | 论文因子(反转/动量)验证 |
| 加密货币 145 币 OHLCV | Binance 公开 API | 2023–2026 | 做空信号验证(extra) |

处理方式:事件研究(PIT 对齐、拆股调整、winsorize、硬弃脏数据)、size 匹配基准、符号检验+Bonferroni。**不含任何个人信息、内幕信息或受限数据**;数据获取符合各来源公开接口的使用条款。已知局限(文档中如实标注):美股样本为存活至今的票(幸存者偏差),这使证伪结论更保守、而非更宽松。

## 7. 运行方式 / 演示方式

**零部署、零 API key、零第三方依赖**(核心引擎纯 Python 标准库):

```bash
python3 src/valkit/lane.py          # 10 项纪律拦截自证(推荐第一个看)
python3 src/valkit/leakage.py       # LLM 泄漏探针:完美回测的"背题者"被当场识破
python3 scripts/run_valkit_pm.py    # 真实数据端到端:Polymarket fade → NO-GO(双死因)
```

演示路径建议(录屏同此):① `lane.py` 看"未预注册就跑→被拒/跑完改门槛→被拒";② `leakage.py` 看 AUC=1.0 的完美策略被探针识破为泄漏;③ `run_valkit_pm.py` 看真实市场数据上的完整判决(含时间衰减表与入场弹跳死因);④ Claude Code 中 `/strategy <一句话想法>` 看 AI 驾驶舱全流程。

## 8. 项目创新点

0. **把策略研究显式建模为进化系统**:变异(LLM 生成)×选择(证伪流水线)×遗传(机制库+教训升级为新闸门)——**连选择压自身都在进化**(开发期间引擎两次从真实数据中发现并修复了自己闸门的缺陷,均有记录)。
1. **"回测=上界"铁律与 LLM 泄漏的工程化处理**:把"LLM 记得历史"从一句担忧变成可测量、可扣除的底噪(strip/corrupt 探针 + cutoff 窗 + PIT 校验)——这是 AI 量化时代验证可信性的关键缺件。
2. **纪律由工具强制而非提示词约定**:冻结先于结果、闸门不可跳、retrofit 被拒绝——LLM 说不动它。
3. **验证层同时是洞察引擎**:每个 NO-GO 附带机制诊断(多空拆解/衰减切片/死因),自动成为下一个假设的线索,构成不自毁的策略自迭代 loop(演示:反转→诊断出多空不对称→派生"只做多输家"→新预注册→仍被时间一致性闸正确杀死)。
4. **引擎自我审计**:32+ 项仪器正控 + 影子登记召回审计(连"引擎会不会误杀"都在量化);开发过程中它自动抓出人工分析漏掉的死因(Polymarket 入场弹跳)并两次纠正自身闸门缺陷,均有记录。
5. **对 RD-Agent 的批判性继承**:完整评估→拆解→排毒记录公开在 docs,展示"如何借开源项目的思想而不继承其方法论缺陷"。

## 10. 风险提示 / 局限 / 未来方向

- **本工具不产生 alpha,只防止自欺**:它提高"说 GO 时为真"的置信度,不保证 GO 会出现;演示中全部 NO-GO 恰是其诚实性的体现。
- 局限:美股演示数据含幸存者偏差(使证伪偏保守);前向确认天然慢,快不了是特性不是缺陷;evaluator 门槛(如成本假设)为保守估计,需模拟盘校准。
- 未来:接入更宽的宇宙(含退市数据)、Kalshi/更多市场适配器、前向 paper 长期运营与召回审计的持续校准、红队 subagent 的常态化。
- **不构成投资建议。**
