# strategy-darwin · 策略的进化机器

> **AI 时代,人人都能让 AI 写出一个"年化 50%"的策略。**
> **没有人告诉你:那个数字,大概率是 AI 编的。**
> 我们造了一台自己挖策略、自己进化、且不会骗你的机器。

---

## 1. 项目名称

**strategy-darwin(策略的进化机器)**
像自然选择一样运转:机器自己大量产生策略想法(**变异**),一台铁面无私的审判引擎负责淘汰(**选择**),每个死掉的策略都留下死因、喂养下一代(**遗传**)。

## 2. 项目简介

这**不是**一个"帮你验证策略"的工具——它是一座**自动运转的策略矿场**:

- **它自己挖**:读论文、翻研报、从历史失败里找灵感,LLM 源源不断提出新的策略假设;
- **它自己进化**:每个被杀死的策略留下"遗书"(死因诊断),机器读着遗书提出下一代——我们亲眼看它从"反转策略死于空头腿"的尸检报告里,自己派生出"那就只做多头腿"的下一代假设;
- **它出奇地诚实**:选择压由确定性程序执行,AI 说不动它。想法可以天马行空,活下来必须货真价实。

和市面所有"AI 炒股"产品的根本区别一句话:**别人向你展示赚钱的回测;我们向你展示 95% 的策略是怎么死的——以及机器如何踩着尸体继续挖。**

## 3. 它解决什么问题

策略研究是一个"想法无限、真货稀缺"的领域。真正的瓶颈有两个,这台机器各吃一个:

**瓶颈一:挖矿太贵。** 读一篇论文、写代码复现、跑完整检验,人工要几天;所以散户只能听消息,机构只覆盖大路货。→ 本机器把"一个想法从提出到判决"压到分钟级,于是可以**海量地挖、不心疼地杀**。

**瓶颈二:挖出来的 99% 是假的,而 AI 让假货更逼真了。** 三种新型骗局:
- **AI 在背答案**——大模型读过整个互联网,包括"历史后来发生了什么"。让它回测,它不是预测,是默写;
- **考完试改及格线**——人和 AI 都忍不住调参数直到"显著",试的次数够多,垃圾也会闪光;
- **纸面利润买不到**——等你真下单,价格、点差、手续费早把利润吃光。

只解决瓶颈一的产品是**幻觉放大器**(挖得越快,骗自己越快);只解决瓶颈二的是安检机(没矿可安检)。**两个一起解决,才是一台能自己转的进化机器。**

## 4. 核心功能:变异 → 选择 → 遗传

**你只需要把想法丢进来——或者什么都不丢,让它自己挖。** Claude Code 里四个命令就是整个产品:

| 你说 | 机器做 |
|---|---|
| `/strategy-loop` | **自挖掘模式**:它读机制库和历史死因,自己提假设、自己实现、自己送审、自己迭代;有幸存者才叫你 |
| `/paper-to-factor <论文>` | 把一篇论文里**所有**策略挖出来(连表格附录里藏的),逐个送审 |
| `/strategy 财报后股价会漂移` | 你的一句话直觉,几分钟变成完整实验与判决书 |
| `/strategy-status` | 仪表盘:挖了多少、死了多少、死因分布、谁还在陪跑 |

**变异(挖)**:三个矿脉——经典论文与研报 / 你的直觉 / **上一代的死因**(最肥的矿脉:每份尸检报告都指向一个更聪明的下一代)。

**选择(审)**:每个候选过一条不可跳关的流水线——先立字据(通过标准提前公证,事后想改?程序拒绝执行)→ **AI 测谎**(抽走全部资料只留代码,还能"预测"对 = 在背答案,作废)→ 时间体检(只在前几年赚钱的幽灵策略,拦)→ 落地体检(晚一天买还赚吗?算全成本还赚吗?资金塞得进吗?)。

**遗传(传)**:死因写进机制库,引导下一代;更狠的是——**教训会升级成新的审判关卡**。开发期间机器已经两次从真实数据里发现自己审判标准的漏洞并当场补上:选择压自身也在进化。

连机器自己也被监督:死刑犯继续纸面陪跑,定期回头审计"有没有错杀好人"。

## 5. 实战记录:48 小时阵亡名单(全部真实判决)

**我们必须坦诚:机器连续运转至今,变异出的每一个策略都被选择压杀死了。24 小时内,我们没有找到一个"有意思"的策略。** 以下是全部阵亡记录:

| # | 策略 | 线索来源 | 死因(人话) |
|---|---|---|---|
| 1 | 预测市场:反着押所有暴涨暴跌 | 人工直觉 | 涨跌混在一起互相抵消,是平均数幻觉 |
| 2 | 预测市场:只反着押暴涨(改良版) | **#1 的尸检报告派生** | 双死因:利润逐年被磨光(2026 只剩零头);且晚一天下单利润蒸发 60%——纸面价差,买不到 |
| 3 | 预测市场:去小盘市场捡错价 | 人工直觉 | 冷门市场反而定价更准;"错价"是没人交易的僵尸报价 |
| 4 | 预测市场:热门偏差、无脑押 NO | 经典文献 | 只在几十美元成交量的鬼盘"成立",真市场归零 |
| 5 | 预测市场:多选项套利 | 人工直觉 | 流动市场概率加总≈1,无利可套 |
| 6 | 预测市场:跟单盈利榜聪明钱 | 外部攻略 | 榜单拆开全是体育专家/速度机器人/超窄玩家,无一可复制 |
| 7 | 美股:短期反转(买输家卖赢家) | 经典论文(1990) | 成本吃光;尸检发现卖赢家一腿被动量碾压 |
| 8 | 美股:只做多输家反弹 | **#7 的尸检报告派生(机器自提)** | 利润全押在 2025 一年,时间体检不过——又一个平均数幻觉,被正确处决 |
| 9 | 美股:12-1 动量 | 经典论文 | 在本样本多空价差只剩 12 个基点,毛利润就不显著 |
| 10 | 加密:冷门币暴跌后做空 | 早期研究复活 | **唯一通过审判的**(命中率 71%)——但落地体检判死:资金费+滑点把微利吃成负数。赢了官司,输在执行 |
| 11 | 韩国散户流:反着买散户抢购的新兴市场 ETF | 模板枚举 | 一段下跌被重复计数;去重后不显著(此判决由机器自动作出——过去需要人工审计才能发现) |
| 12–15 | 韩国散户流:杠杆 ETF / 大盘杠杆 / 个股,反着做与接盘,共 5 条 | 模板枚举 | 分别死于:那是动量不是反转 / 连续 7 年方向反指 / 跨年扩样归零的时代幻觉 |
| 16 | 韩国散户流:LLM 自创的 5 个 ETF 反转 | **LLM 自挖掘** | 提了数据里不存在的品种——此漏洞已修复:现在生成前先看"矿里有什么" |
| 17 | 韩国散户流:极端买入后必反转 | 人工直觉(项目起点) | 事实相反:极端买入后 5 日平均**上涨** 1.1% |

**约 20 条策略,横跨 3 个市场、4 种线索来源(论文/人工/LLM 自挖/尸检派生),全部阵亡,每条都有可复查的判决书**(仓库 `runs/` 与 `docs/`)。

我们认为这份名单恰恰是本作品最有力的证明:
- 它证明机器**真的在挖**(四种矿脉都出过矿,包括机器自己派生的第二代);
- 它证明机器**真的不放水**(连"通过审判"的那一个,都被落地体检拦了下来);
- 而一台连续 48 小时不肯说一句假话的机器,比一百个喊"年化 50%"的机器人值钱。**哪天它说"通过",那句话才有分量。**

## 6. AI 用在哪

一句话:**AI 是这台机器里最能干的员工,但永远不是法官。**

- **AI 当矿工**:读论文、结合机制库与历史死因,提出下一代假设(Claude 智能体工作流 + 跨会话记忆);
- **AI 当工程师**:自然语言想法几分钟变成可运行实验(Claude Code 智能体 + 工具调用 + 自动化工作流);
- **AI 当红队**:独立 AI 分身专门攻击"通过"的判决(多智能体对抗);
- **AI 永远不当法官**:判决由确定性程序做出,AI 说不动它;
- **AI 还是嫌疑人**:我们可能是极少数**给 AI 配测谎仪**的量化项目——在金融历史数据上,大模型"过目不忘"恰恰最危险。

(工程:Claude Code 承载智能体流程;实验中另用 DeepSeek/豆包 API;审判引擎为自研纯 Python,零第三方依赖。)

## 7. 数据来源与处理

全部**公开数据**,随仓库附带(~36MB),无个人信息/内幕信息/受限数据:预测市场 **7,718 个已结算市场**(Polymarket 公开 API,2022–2026)、美股 **235 只**日线(Stooq/Nasdaq,2020–2026)、加密 **145 币**(Binance 公开 API,2023–2026)。标准事件研究处理(时点对齐、除权、去极值);已知局限如实标注(美股样本幸存者偏差——只会让审判更严,不会更松)。

## 8. 怎么体验(评委 60 秒)

零部署、零账号、零 API key、零第三方依赖:

```bash
git clone https://github.com/york-zhouuu/strategy-darwin && cd strategy-darwin
python3 src/valkit/lane.py       # 看点①:想跳过公证、想事后改标准 → 程序当场拒绝
python3 src/valkit/leakage.py    # 看点②:回测满分的"天才策略"被测谎仪揭穿:它在背答案
python3 scripts/run_valkit_pm.py # 看点③:7718 个真实市场,阵亡名单 #2 的完整判决书现场重演
```

配套录屏演示同一路径 + Claude Code 驾驶舱自挖掘实操。

## 9. 创新点

1. **验证器同时是创意引擎**:死因诊断自动成为下一代策略的进化方向(阵亡名单 #2、#8 就是机器读尸检报告自己提出的第二代)——挖掘与审判在同一个闭环里,越用越聪明。
2. **给 AI 配测谎仪**:业内都在担心"大模型回测作弊",我们把担心做成了可运行的检测——抽走资料还答对 = 背答案,当场揭穿。
3. **法庭不许翻案**:通过标准在看结果前被程序锁死,事后修改被代码层拒绝——科研级"预注册"第一次成为量化工具里绕不过的机关。
4. **审判者也受审**:死刑犯纸面陪跑 + 定期错杀审计;开发期间机器两次抓出人工漏掉的死因、两次修复自己标准的漏洞,全程账本可查。
