# valkit × RD-Agent:生成器 + 裁判 的有机结合

> 一句话:**RD-Agent 是 D(把论文/研报快速变成可跑的因子代码),valkit 是 R(拒绝相信,直到它活过杀戮道)。** 二者相加 = 工业化地证伪因子文献。
> 关联:`tradeagent4/RD-Agent评估.md`(RD-Agent 是好 D 层、坏 R 裁判)、`docs/valkit-验证基建总览.md`。

## 为什么天生互补

| | RD-Agent 单独 | valkit 单独 | 结合 |
|---|---|---|---|
| 生成候选 | ✅ 快(读论文→实现因子) | ❌ 每个手写 | ✅ RD-Agent 供吞吐 |
| 判真伪 | ❌ 反馈=回测 IC/ARR=过拟合机器 | ✅ 强制杀戮道、不被骗 | ✅ valkit 当裁判 |
| 净效果 | 快速把自己骗过去 | 严谨但候选靠手搓 | **高吞吐生成 + 高完整性证伪** |

RD-Agent 评估里那句"当 D 层工具、外面套你自己的 R 裁判",这里落成具体接口。

## 一条不踩就中毒的红线

**valkit 的 verdict 绝不能回喂进 RD-Agent 的迭代优化环。**
否则 RD-Agent 从"过拟合 IC"变成"过拟合通过 valkit"——**还是 p-hacking,只是换了靶子(C3 设计过拟合)**。正确姿势 = 生成/裁判分离,对应置信阶梯的 ③研究员留出:

- RD-Agent 是"研究员",在 **DEV 窗**自由迭代(它自己的廉价内部筛随便用)。
- 产出的因子**冻结配置+哈希**,在 **TEST/最近期/OOS** 上**一次性**走 valkit 杀戮道。
- 若要拿 valkit 结果指导下一轮 RD-Agent → 每轮记一次多重检验罚(看数预算),TEST verdict 一次定生死。

## 一个不显然的泄漏点

RD-Agent 产的因子公式**本身是机械的**,但**"选哪个因子来实现"是 LLM 读了(已记过的)论文选出来的 → 选择被 C1/C2 污染**。故:

> RD-Agent 的因子 MUST 走 valkit 的 **agent 路径**(`is_agent=True`,过 `leakage`/`cutoff`/`pit` 审计),不能当纯机械 thesis。污染在**选择**里,不在计算里。

## 推荐接法:只用"实现"、不用"回测"(精简且干净)

RD-Agent 危险的是 Qlib 回测驱动的迭代;有价值的是 `paper→factor code`(general_model / factor_from_report 的抽取+CoSTEER 实现)。

**只取后者,整条 Qlib/CSI300 + 毒反馈环一刀切掉**,valkit 在你自己的数据/宇宙上当唯一裁判 → 一步躲开毒环 + A股耦合 + 大半 Docker/Qlib 重装。

```
论文/研报 ─(RD-Agent: 抽取+CoSTEER 实现)→ 因子代码 + 每股每日因子值
                                              │
                       valkit/adapters/rdagent.py:因子值 → Event(t0=调仓日, ret=前向收益)
                                              │
                    KillLane.run(is_agent=True, leakage/cutoff/pit=…, tradeability=…)
                                              │
                             NO-GO(≈95%,归档) / 幸存者 → 前向 paper(懒建)
```

## 待建的接口(一个 adapter)

`valkit/adapters/rdagent.py`:把 RD-Agent 一个因子的时序输出(universe × date × 因子值)转成 valkit 契约——
- **Event**:决策点(如每次调仓日的因子极值分位),`t0`=调仓日,`bucket`=流动性/板块桶。
- **ret(e,h)**:该分位组合的前向 h 期净收益;可选 `ret_delayed`(入场弹跳)、`side`(多空)。
- **agent 审计入参**:`cutoff_audit`(声明生成所用模型的 cutoff)、`pit_check`(因子输入是否 PIT)、`leakage_audit`(无输入/反事实探针)。

## 用途(这套结合是干什么的)

**工业化地证伪因子文献。** 具体:

1. **把海量文献变成可跑候选**——因子动物园(~数百个已发表因子,多数不复现或已被套干)、任意论文/研报,RD-Agent 廉价实现,不用手搓每一个。
2. **正确地杀死其中 ~95%**——valkit 当"复现危机探测器":对每个因子问"扛不扛得住 OOS / 时间衰减 / 成本 / 泄漏 / 小资金可交易",且**不被 RD-Agent(或论文)的过拟合与泄漏骗**。输出以 NO-GO 为主(这正是它的本职——它是 killer)。
3. **罕见的幸存者 → 真线索**,进(懒建的)前向 paper / 确认道。

这也**解掉了早先的顾虑**:单独做"自挖掘因子(path 2)"是 p-hacking 机器;但 **RD-Agent(生成)+ valkit(裁判)分离**后,自动挖掘变安全了——因为裁判是不可迭代、不被骗的那道闸。

## 策略 loop:验证层当洞察引擎(不只是闸)

valkit 的诊断(`by_bucket` 多空拆 / `by_period` 衰减 / `a'` 成本·弹跳 / 泄漏)让**每个 NO-GO 都附带一个"为什么死"的机制线索,那线索就是下一个策略**。这把过滤器升级成**研究引擎**,也是它胜过 RD-Agent 之处:RD-Agent 的反馈是一个傻标量(IC),valkit 的反馈是**结构化机制诊断**。

```
       ┌───────────────────────────────────────────────┐
       │  RD-Agent / LLM(D):论文 → 因子代码             │
       ▼                                                 │ 机制洞察(非旋钮)
  valkit 杀戮道(在发现集 D 上)──NO-GO 95% + 诊断 ────────┘
       │ 幸存者(罕见)
       ▼
  确认集 F(新/前向,loop 从未挖过)—— 前向 paper,唯一能宣布赢家处
```

### 一条不踩就中毒的红线(loop 版)

feedback 有两种,一种是毒:

| 类型 | 例子 | 性质 |
|---|---|---|
| **旋钮追分(毒)** | "5 日反转没过,试 3/7/21 日直到过" | 同数据上朝"通过裁判"优化 → 过拟合裁判,GO 变废纸(C3) |
| **机制洞察(合法)** | "赢家动量毒了反转空头腿 → 试只做多输家" | 关于世界的新假设,可独立检验 |

即使是机制洞察,loop 仍需**跨迭代数据卫生**,否则"同数据上生成洞察又确认"= 跨迭代 p-hacking:
1. **洞察在发现集生成,确认在没碰过的新/前向数据**;loop 可在杀戮道侧快速空转(廉价生成、杀 95%),但**收敛(宣布赢家)只能在前向 paper**——这就是确认道存在的理由(loop 的诚实锚)。
2. **全 loop 记多重检验**:`append-only 账本`天然是 **shot-counter**(每次 `register` 一枪);任何最终 GO 的显著性 MUST 按总枪数折价,否则"打够多枪总有一个过"。

### 已端到端演示(2026-07-04,我当 RD-Agent 的 D 层,数据 tradeagent3 本地美股 CSV)

- **第 1 圈** `scripts/run_valkit_reversal.py`:论文因子**短期反转** → 🔴NO-GO(死于成本);a' 多空拆诊断出"赢家动量毒空头腿、输家反弹 +63bps"。
- **第 2 圈** `scripts/run_valkit_loserbounce.py`:从该洞察派生"**只做多输家**",**开新 prereg**(非 retrofit)诚实再验 → 过了 a'(可交易/扛成本/不弹跳),**但栽在时间一致性**(收益全在 2025、2026 不显著)→ 🔴NO-GO。
- **结论**:旋钮追分会"找到"那 +63bps;**有纪律的 loop(`require_latest_period`+signZ+OOS)正确杀死了 pooled 幻觉** → 证明 loop 不自毁。账本已记 7 shot。

> **关键澄清**:机械计算的因子(即使公式来自论文/LLM)走**机械路径**;"论文=已知因子"的选择性污染由 OOS/最近期闸防御,不是靠泄漏探针(泄漏探针是给逐事件 LLM 定性判断的 agent 路径)。

## 诚实的边界

- 它**过滤,不创造 edge**。有没有幸存者由生成质量 + 战场决定;流动拥挤因子上,预期全 NO-GO(edge↔可交易性墙照旧)。
- **确认侧仍懒建**:某因子活过上界只是"值得前向",不是"能赚钱";净收益由前向 paper 定。
- RD-Agent 即便只用实现层,仍需 Python≥3.10 + 付费 LLM key(见评估文档);valkit 侧零依赖。
