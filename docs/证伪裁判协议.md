# 证伪裁判协议（Falsification Judge Protocol）

> 跨战场复用的统一入口。定位：**裁判是耐用资产,策略/数据/战场都是一次性可丢的。** 任何新 thesis(股/币/预测市场/资金流/论文因子……)都从这里进,走同一套判决,产出同格式的 go/no-go + 归档。
> 版本 v0.2 · 2026-07-04 · 关联 `L1-latency-investigation.md`(edge↔可交易性墙)、RD-Agent 评估(`tradeagent4/RD-Agent评估.md`)。
> **代码实现**:③A 机械回测半 + 预注册 + 报告已落成 `src/valkit/`(战场无关,adapter-per-战场);冒烟测试 `scripts/run_valkit_pm.py` 复现 Polymarket P0。③B 前向 paper 半(`valkit/paper.py`)待建。

---

## 0. 为什么要它

四个项目的教训:死因不是"测得不够准",是①**地形**(有 edge 的角落交易不了)②**泄漏**(agent 的价值恰恰回测不了)。所以裁判必须先管住这两件,再谈统计。基建是放大器不是瓶颈——**它不产 hypothesis,只让证伪更快更诚实。**

判决顺序是**四道闸,前闸不过不进后闸**:

```
① 战场准入闸(纸上) → ② 泄漏路由 → ③A 机械回测半 / ③B 前向paper半 → ④ 判决与归档
```

---

## ① 战场准入闸(投一行代码前,先在纸上过)

thesis 要能**同时**占齐 4 条,缺一角就是前三个项目的死法,别建管道:

- [ ] **真锚**:价格有"该去的地方"(基本面/真实概率/结算值)。无锚=测的是趋势延续不是收敛。
- [ ] **够冷门**:结构 edge 在(没被量化盯死)。最被盯的资产(BTC/大盘)这一条当场蒸发。
- [ ] **agent 可处理**:信息脏、杂、量大、非结构化——LLM 的主场,否则纯机械就够,agent 无增量。
- [ ] **小资金能真交易**:$10k–100k 进得去、出得来、成本吃得起。极端不流动角落这条常常否掉。
- [ ] **可验证**:agent 在此能产出足够多、够快结算的独立预测,使前向证伪在合理时间内可达。慢催化战场即使有 edge 也验不出 → 降优先级(与 edge 是否存在无关)。见 `openspec/changes/define-agent-validation-protocol/`。

> 5 条**全绿**才进。任一条纸上就红 → 记一句话理由,归档,不建管道。

---

## ② 泄漏路由（进来先分流,这是协议的核心)

**泄漏铁律**:LLM 训练数据含未来。判据——

| 这个环节的判断是否依赖"LLM 对历史事件的定性/文本推理"? | 路由 |
|---|---|
| 否(纯数值、point-in-time 可复算) | → **③A 机械回测半**(可干净历史回测) |
| 是(读新闻/研报/叙事、跨源定性拼图) | → **③B 前向 paper 半**(只能前向,历史回放结果一律不作结论) |

> 一个 thesis 常被拆成两段:机械筛子走 A,agent 判断走 B。**别让 A 的干净回测数字给 B 的泄漏结论背书。**

---

## ③A 机械回测半（已建,formalize 成清单）

引擎已在 `src/tradeagent/`;每跑一次留痕 `runs/journal.jsonl`。判决清单:

- [ ] **预注册**(见附录模板):看数据前钉死 go/no-go 门槛,冻结到独立配置。
- [ ] **仪器正控**:先向合成数据注入已知漂移,确认引擎测得回。**用它证伪前,先证明它能测出真信号。**
- [ ] **基准**:异常收益 = 个股 − **size/流动性匹配基准**(非大盘指数,避免把 size 溢价错当 edge)。
- [ ] **调整价**:拆股/分红调整,硬弃 |收益|>300% 脏数据,winsorize。
- [ ] **方向性度量**:`sign(初始反应)×后续异常收益`(无条件平均对称事件自抵消至≈0,是假象)。
- [ ] **入场无前视**:t=0 严格之后首个可交易时点;**测入场时点敏感性**(收盘 vs 开盘 vs 跳N日),分辨真 edge 还是买卖价差弹跳。
- [ ] **净成本**:保守成本模型后仍要有肉。
- [ ] **样本内发现 / 样本外确认**:只认 OOS 仍成立的规则。
- [ ] **时间衰减切片(一等检验,别跳)**:按年/期切,看 edge 是否被套利磨平。⚠️ **中位数 OOS 会把强旧段和弱新段混进一个桶、掩盖衰减**——须额外要求**最近一期单独过 Bonferroni**(Polymarket P0 混年看 signZ+8,按年切 2026 掉到+2.1;加固前给假 GO)。
- [ ] **多重检验**:分层×分位×窗口的所有检验做 Bonferroni;肥尾下同时看均值 t 和符号检验 z。
- [ ] **可交易性拆解**:把方向性拆成多/空原始收益——稳的那半是不是只能做空?能做多的那半是不是靠幸存者灌水的肥尾?

## ③B 前向 paper 半（缺的那半 · 最小设计)

> **深度展开见 `openspec/changes/define-agent-validation-protocol/`**:把泄漏拆成 5 通道(C1–C5)、判断主张(a)先于策略主张(b)、5 级置信阶梯(机械地板→泄漏审计回测当上界→研究员留出→前向 paper→skill 归因)。**核心结论:agent 回测一律当上界,置信只在前向 paper 赚取。** 本节是其中④的最小落地。

对 agent/LLM 环节唯一合法的验证。核心=**预测必须在结局之前、不可篡改地落地**。三个部件:

**1. 预测登记簿 `runs/predictions.jsonl`(append-only,写时对结局全盲)**
决策时点(t=0)每条写入,之后不可编辑:
```jsonc
{ "ts": "<t=0 UTC>", "thesis_id": "...", "instrument": "...",
  "prediction": { "direction|prob|target": ... },   // agent 的判断
  "confidence": 0-1,
  "inputs_snapshot": "<agent 当时看到的原始输入,或其哈希>",  // 防事后说"我其实看到了别的"
  "exit_rule": { "hold_days": N | "target": ... | "stop": ... },  // 预先钉死,不许事后改
  "price_at_t0": ... }
```

**2. 结算作业(独立进程,晚于 t0+N 才跑)**
读 t0+N 的价格/结局,回填 `outcome`。**评分器在预测时点绝不运行**——这是防泄漏的物理隔离。

**3. 评分(与回测半同套统计,只是前向累积)**
- 概率型(Polymarket):Brier + 校准曲线。
- 方向型(股/币):命中率、净成本后均值(vs size 匹配基准)、符号检验 z。
- **预注册**:开跑前钉死"需 ≥N 条预测、跨 ≥M 周,PASS 门槛 = Brier<X / 命中>Y(净成本后)";**样本量不够不出判决**。
- 正控类比:掺入几条已知易判的预测,确认 harness 打分正确。

> 铁律:**任何历史回放里 agent 参与的结果都不算数**,只有这套前向登记簿累积的样本能给 L3/L4 判决。

---

## ④ 判决与归档

- 产出统一格式 go/no-go 报告(判定表 + 覆盖标的 + 样本量 + 被截断/丢弃数据 + 已知局限),模板见 tradeagent3 `reports/validation_report.md`。
- **绿灯** → 记录进入下一层(A 过 → 启动 B 前向;B 过 → 才谈实盘)。
- **红灯** → **干净止损,归档"此路已证伪 + 原因"。不在被证伪的结论上打补丁**;换标的/换信号=新 change,不是补丁。

## 附录:预注册模板(看数据前填,冻结)

```
thesis_id:            <一句话假设,含方向>
战场准入 4 条:         真锚[ ] 冷门[ ] agent可处理[ ] 小资金可交易[ ]
路由:                 机械回测 / 前向paper / 两段(哪段走哪)
主假设(唯一 go/no-go): <预测什么 → 收益/校准应如何>
过关门槛(钉死数值):     方向/深度<___ · 显著性 p<___ 或 signZ>___ · 净成本后 · 跨标的一致≥___
样本量下限:            机械≥___事件 / 前向≥___预测且≥___周
基准:                 size匹配 / 无条件均值(对照)
多重检验:              主假设唯一判据,其余持有期/分层/方向=探索性
证伪即止条件:           <什么结果出现即宣布死亡、不再找补>
```

> 一句话:**裁判管四道闸——先问地形(准入)、再分泄漏(路由)、机械走回测半、agent 走前向半、最后统一判决归档。喂任何新 thesis 都走这一条。**
